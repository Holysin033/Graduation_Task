# 基于Uni-app和Node的音乐听歌系统答辩问题

## 1. 技术选型相关问题

### 问题1.1
**Q**: 为什么选择 Uni-app 而不是其他跨平台框架（如 Flutter）？在开发过程中遇到了哪些跨平台适配的挑战？

**提示**：
- Uni-app 基于 Vue.js 生态，学习成本较低
- 支持多端编译，开发效率高
- 遇到的挑战：
  - 音频播放器在不同平台的兼容性问题
  - 样式适配问题
  - 组件库的兼容性问题

## 2. 系统架构相关问题

### 问题2.1
**Q**: 系统的前后端分离架构是如何实现的？为什么选择 Node.js 作为后端技术栈？

**提示**：
- 前端架构：
  - Vue.js + Uni-app 构建
  - 组件化开发
  - 状态管理（Vuex）
- 后端架构：
  - Node.js + Express 框架
  - RESTful API 设计
  - 数据同步机制
- Node.js 优势：
  - 高性能
  - 异步 I/O
  - 适合实时应用

## 回答建议

1. 准备充分：对每个问题都要有深入的理解和具体的实现细节
2. 结合代码：在回答时最好能结合具体的代码实现
3. 突出重点：强调系统的创新点和特色功能
4. 实事求是：承认系统的不足，并说明改进方向
5. 逻辑清晰：回答要有条理，层次分明
